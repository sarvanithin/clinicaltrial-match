About ClinicalTrial Match

Open-source AI matching between patients and clinical trials

Privacy First

Patient data is never stored. Your FHIR bundle or clinical note is processed entirely in server memory and discarded immediately after matching. No patient records, no logs, no retention.

How It Works

  1. Upload FHIR bundle or clinical note
  2. AI extracts age, gender, diagnoses, labs, medications
  3. Semantic search across 400K+ trials from ClinicalTrials.gov
  4. Rule-based eligibility constraint evaluation
  5. Composite score (40% semantic + 60% constraints)
  6. Ranked results with plain-English explanations

Open Source

Self-host with your own API key. Full source on GitHub.

Data Sources

Trials synced from ClinicalTrials.gov API v2 (public domain). Only trial metadata is cached — no patient data ever touches our database.
